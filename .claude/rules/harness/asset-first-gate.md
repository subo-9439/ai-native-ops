# 에셋 우선 게이트 (asset-first-gate)

> [PM 적용 제외] project-manager=운영 인프라(Node/스크립트), Flutter UI 없음. SSOT 일관성 위해 복제하되 본 규칙은 game(whosbuying)에만 강제.

## 원칙

**이모지/임시 이미지를 코드·UI·아이콘에 쓰기 전, 반드시 프로젝트 이미지 에셋을 먼저 검색한다.**
이모지나 즉석 SVG 생성은 **에셋이 없을 때만** 허용한다.

## 적용 대상

다음 작업에서 이모지·즉석 그래픽을 추가하려 할 때:
- favicon, app icon, og-image 등 브랜드 노출 영역
- 게임 카드/배지/뱃지/카테고리 라벨
- 랜딩, 로비, 메뉴, 홈 화면 등 사용자 노출 UI
- 스플래시/로딩/엠프티 스테이트
- 마케팅·SNS·문서 썸네일

## 검색 절차 (필수 3단계)

1. **에셋 디렉토리 목록**:
   - `game_project_app/assets/icons/` (brand, games, common)
   - `game_project_web/web/` (favicon, og-image, manifest icons)
   - `.local-docs/claude_design_dump/` (디자인 덤프)
2. **키워드 검색**: `Glob` 또는 `find`로 의도한 개념 키워드(예: dice, bomb, ladder, logo) 매칭 파일 탐색.
3. **없을 때만 fallback**: 검색 결과가 없거나 명백히 부적합한 경우에 한해 이모지/즉석 SVG 사용. 이때도 "에셋 없음 확인 완료"를 답변에 명시한다.

## 위반 사례 (실제 발생)

- 2026-04-20 favicon 교체 작업: 직접 5점 SVG를 그려 커밋·배포함. 이미 `assets/icons/brand/logo_dice.svg`에 브랜드 다이스 로고가 존재했음. 결과: 즉시 재교체 + 재배포 비용 발생.

## 검증 표현

에셋 우선 검토를 마쳤다면 답변에 한 줄로 명시:
- ✅ `에셋 검색 완료: <경로> 사용` (에셋 발견 시)
- ✅ `에셋 검색 완료: 매칭 없음, 이모지/즉석 그래픽 fallback 사용` (없을 때)

검증 표현이 없으면 이 규칙을 무시한 것으로 간주한다.

## 예외

- 제품 외부 컨텍스트(README 임시 표기, 커밋 메시지, 채팅 답변 등)는 적용 제외.
- 단, 사용자 화면에 노출되는 모든 텍스트/UI는 본 규칙 적용.

## 위반 시

- 사후 발견 시: 즉시 에셋 기반으로 재교체 후 사용자에게 사고 보고.
