# 의도 추론 게이트 (intent-inference-gate)

> **rules만으로는 충분하지 않다.** 이 규칙은 Discord 봇 페르소나 (DEV_CONTEXT [의도 추론
> 의무] 섹션) + 답변 검증 표현 강제와 함께 동작한다.
> 재발 사례: 2026-05-03 PR-SOLO3 — "솔로 모드 = 봇 기반" 명세를 받고 봇 spawn API 그대로
> 호출 → 결과 패널에 "봇-01" 노출. 누구나 자연스럽게 떠올릴 변형 (이름 변경, 페르소나
> 부여) 같이 제안하지 못함.
> CEO "이건 기획 문제냐 개발 문제냐?" 보고. 답: 둘 다.

## 원칙

명세를 받았을 때, **표면 명세** 와 **사용자 의도** 를 항상 분리해 인식한다.
단순 implementation 은 의도 위반의 흔한 패턴.

누가살래는 **"오프라인 모임에서 누가 살지 결정하는 캐주얼 파티게임"** — 사용자는 "동작"
보다 "재미 / 자연스러움 / 한 판 더" 를 본다. 표면 명세 ("X 를 Y 로 처리해") 의 뒤에 있는
의도 ("이런 느낌이 나게") 를 추론하지 않으면 결과가 평균 이하다.

## 4가지 의무 (모든 명세에 강제)

| # | 의무 | 검증 방법 |
|---|------|----------|
| 1 | **표면 명세 vs 사용자 의도** 분리해서 명시 | 답변에 "표면: ..., 의도: ..." 또는 동의어 표현 |
| 2 | **자연스러운 변형 1~3개** 같이 제시 (사용자가 명시 안 했어도) | 답변에 "변형 가능성: 1) ... 2) ... 3) ..." |
| 3 | **공통 도메인 추출 가능성** 점검 | 비슷한 책임이 있으면 "공통 추출: <어디 어디 묶을 수 있음>" |
| 4 | **변형 차단 시 명시 사유** | 단순 implementation 이 정당한 경우(테스트, prototype, hotfix) 명시 |

## 답변 검증 표현 (필수)

CEO/사용자 명세를 받은 응답에는 **둘 중 하나** 가 반드시 명시되어야 한다:

- ✅ `의도 추론 완료: 표면=<...> / 의도=<...> / 변형 N개 / 공통 추출=<있음/없음>`
- ⚠️ `의도 추론 생략: 사유=<테스트/prototype/hotfix>`

명시가 없으면 본 규칙을 무시한 것으로 간주한다.

## 적용 대상 (필수 게이트 통과)

다음 명세를 받을 때 4가지 의무 강제:

- 새 기능 / 화면 / 게임 모드 추가 (예: "솔로 모드", "AI 라이어", "관전 모드")
- 새 BE 엔드포인트 / 봇 / 서비스 추가
- UI 패턴 추가 (예: "결과 화면에 공유 버튼")
- 시각 토큰 / 라벨 / 색상 변경 (connected-work-gate 와 같이 적용)
- B2B 임베드 / 새 폼팩터 대응

## 적용 제외

- 단순 typo / 주석 / 포맷팅 수정
- 단일 commit hotfix (사유 명시 시)
- 명백한 정책 작업 (예: "테스트 추가", "import 정리")
- 외부 종속성 / 도구 변경 (CLI 옵션 추가 등)

## 다른 게이트와의 관계

## 이중 방어 원칙

| 계층 | 도구 | 역할 |
|------|------|------|
| L1 — 페르소나 | Discord 봇 DEV_CONTEXT [의도 추론 의무] (claude.js) | 모든 dev role 에 자동 적용 |
| L2 — 답변 검증 | "의도 추론 완료" 표현 강제 | 누락 즉시 사용자 감지 |
| L3 — 회귀 사례 | 본 게이트의 위반 사례 표 | 회귀 시 인용 → 같은 실수 차단 |

세 계층 중 하나라도 거부하면 해당 작업은 수행하지 않는다.

## 위반 시

- L1: Claude 자발적으로 다음 응답에 자가 보고 + 후속 fix
- 사후 발견 시: 의도 추론 결과 + 변형 제안 + 공통 추출 안을 새 메시지로 즉시 추가 + 위반 사례 표 갱신
