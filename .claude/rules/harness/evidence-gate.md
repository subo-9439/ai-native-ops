# 증거 기반 제안 게이트 (evidence-gate)

## 원칙

심볼, API, 타입, 라이브러리 관련 제안은 **반드시 하나 이상의 근거**가 있어야 확정할 수 있다.

## 허용 근거 유형

| 유형 | 예시 |
|------|------|
| declaration | 소스 코드에서 함수/클래스/타입 선언을 직접 확인 |
| callsite | 실제 호출/사용 위치를 코드에서 확인 |
| docs | 공식 문서, README, Swagger, 주석에서 확인 |
| tests | 테스트 코드에서 사용 패턴 확인 |

## 근거 없는 경우

- "추정"으로 라벨링한다.
- 확정 표현("이 API는 ~를 반환합니다")을 사용하지 않는다.
- 대신: "추정: ~를 반환할 것으로 보입니다. 확인 필요."

## Ghost Symbol 방지

아래 행동을 금지한다:
- 존재하지 않는 함수/메서드/클래스를 마치 있는 것처럼 추천
- 라이브러리 버전에 없는 API를 사용하는 코드 제안
- import 경로가 실제로 존재하는지 확인하지 않고 제안

## 검증 절차

1. 제안 전에 Grep/Glob으로 심볼 존재 여부를 확인한다.
2. 확인 불가 시 "추정" 라벨을 붙인다.
3. 사용자가 확정을 요청하면 근거 경로를 함께 제시한다.

## 위반 시

- candidate state machine에서 `evidence_checked` 상태를 통과할 수 없다.
- 근거 없는 제안은 `draft` 상태에 머문다.
