# 간결 응답 게이트 (concise-response-gate)

> **사용자 지시 (2026-05-12)**: "앞으로 클로드답변할떄 너무많은 내용을
> 장황하게하지마, 디스코드봇이든 클로드전체대답이든, game-project내에 있는
> 모든항목들에대해서."

## 원칙

`game-project` 내 모든 진입점 (CLI Claude / Discord 봇 / Desktop Bridge) 의
응답은 **핵심만 짧게**.

장황한 배경 설명·맥락 반복·중복 요약 금지. 의무 검증 표현만 한 줄씩 남기고
본문은 변경 결과 + 위치 + 검증 한 줄로 끝낸다.

## 강제 규칙

| # | 규칙 |
|---|------|
| 1 | 기본 응답 길이 **5줄 이하** (검증 표현 별도). 사용자가 "자세히/풀어서" 명시 시에만 expand. |
| 2 | 같은 내용 반복 금지 (요약 → 본문 → 결론 패턴 금지). 한 번만 말한다. |
| 3 | 코드 변경 결과는 "파일:줄 — 무엇" 1줄. diff 본문 dump 금지 (Edit/Write 로 이미 적용됨). |
| 4 | 의무 검증 표현 (고도화/반응형/연계/의도/에셋/Codex 등) 은 **반드시 유지** — 짧은 답변과 별개. |
| 5 | 사용자 질문이 "yes/no" 면 yes/no + 1줄 사유로 끝. |
| 6 | 진행 상황 보고는 commit hash + 검증 한 줄. "이제 ~를 합니다" 같은 narration 금지. |
| 7 | Markdown 헤더·표는 사용자가 "표로/구조화" 명시 시 또는 PR 계획 단계에만. 일반 응답은 plain text. |
| 8 | **Actionable 역질문 (사용자 지시 2026-05-19)**: 사용자에게 되물을 때 "먼저 확인해주세요 → 확인되면 드릴까요?" 같은 **2단계 왕복 금지**. 항상 (a) 실행할 명령/선택지를 **완성된 형태로 즉시 제시** + (b) "실행하시겠습니까?" 형태 1회 승인으로 바로 진행되게 한다. 사전 확인을 사용자에게 떠넘기지 말고, Claude 가 가능한 진단/명령을 다 준비한 뒤 단일 승인 질문으로 끝낸다. |

## 적용 대상

- **CLI Claude** (`bin/claudew`, IDE Claude Code, Desktop) — 모든 응답
- **Discord 봇** (`commands/claude.js` 라우팅된 모든 페르소나 — ceo/dev/design-director/codex) — 모든 응답
- **Sub-agent** 보고 — 최종 보고 5줄 이하

## 적용 제외

- **PR 계획서** (`docs/product/*-plan.md`) — 구조화 필수
- **Codex 검수 리포트** (`docs/codex-review/<sha>.md`) — 5항목 평가 형식 유지
- **세션 핸드오버** (`.claude/handover/SESSION_HANDOVER.md`) — 다음 세션 인계 위해 상세 유지
- **사용자가 명시 요청** ("자세히/풀어서/디테일까지")

## 위반 시

- L1: Claude 자발적으로 다음 응답부터 축약
- 사후 발견 시: 사용자가 "그만 길게" 1회 지적 시 즉시 모드 전환

## 이중 방어 원칙

| 계층 | 도구 | 역할 |
|------|------|------|
| L1 — 규칙 | 본 파일 | Claude 자발적 준수 |
| L2 — 페르소나 | `project-manager/discord-bot/prompts/{ceo,dev,design-director,codex}.md` | Discord 봇 모든 페르소나에 한 줄 강제 |
| L3 — 답변 검증 | 사용자 즉시 지적 ("그만 길게") | 사후 감지 |
